# AI 会话导出工具

一个私人使用的浏览器扩展，能够从 AI 会话网页中导出内容为 Markdown 格式。

## 支持的平台

- ✅ ChatGPT (chatgpt.com)
- ✅ Claude (claude.ai)
- 🔜 其他平台（通用提取模式）

## 功能特性

### 核心功能

1. **区分用户与 AI 内容** - 自动识别并分别导出用户提问和 AI 回复
2. **Markdown 格式导出** - 保留原始结构（标题、段落、列表、代码块、表格等）
3. **引用提取** - 自动提取 AI 引用文献或网址
4. **思考过程** - 如果 AI 展示了思考/推理摘要，会一并提取

### 导出选项

- 包含时间戳
- 包含模型信息
- 包含引用来源
- 包含思考过程（如果有）

## 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                       浏览器扩展                              │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   popup.js  │  │ background.js│  │    content.js       │  │
│  │  (UI 界面)   │  │ (Service Worker)│  │  (内容脚本)          │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
│                              │                               │
│                              ▼                               │
│                    ┌──────────────────┐                      │
│                    │   injected.js    │                      │
│                    │  (页面注入脚本)   │                      │
│                    │ - Fetch 拦截      │                      │
│                    │ - XHR 拦截        │                      │
│                    │ - 页面状态读取    │                      │
│                    └──────────────────┘                      │
└─────────────────────────────────────────────────────────────┘
```

### 三层数据采集策略

1. **网络层拦截**（首选） - 拦截 `fetch` / `XMLHttpRequest`，从 API 响应获取原始数据
2. **页面状态读取**（辅助） - 读取页面内嵌的状态数据
3. **DOM 解析**（兜底） - 从可见 DOM 结构提取，当网络层失败时使用

## 安装方法

### 开发模式安装（推荐用于测试）

1. 打开 Chrome/Edge 浏览器
2. 访问 `chrome://extensions/` (Chrome) 或 `edge://extensions/` (Edge)
3. 开启右上角的 **"开发者模式"**
4. 点击 **"加载已解压的扩展程序"**
5. 选择本项目的文件夹

### 加载后

扩展图标会出现在浏览器工具栏中，点击即可使用。

## 使用方法

1. 访问 ChatGPT 或 Claude 的会话页面
2. 点击浏览器扩展图标
3. 确认检测到正确的平台
4. 选择导出选项
5. 点击 **"导出当前会话"**
6. Markdown 文件将自动下载

## 注意事项

1. **本地优先** - 所有数据处理都在本地完成，不会上传到任何服务器
2. **私人使用** - 仅供个人学习研究使用
3. **网络请求拦截** - 扩展需要拦截页面网络请求以获取数据，这是正常功能
4. **平台更新** - 如 ChatGPT 或 Claude 大幅改版，可能需要更新提取逻辑

## 技术说明

### 关于"思考过程"提取

- 只能提取**对用户可见**的思考摘要/推理摘要
- 无法获取模型内部完整的 chain-of-thought
- 如果界面没有展示思考过程，则不会包含此内容

### 数据来源边界

- ✅ 提取用户界面上可见的数据
- ✅ 提取网络响应中公开的数据
- ❌ 不尝试获取未公开的隐藏内部字段
- ❌ 不做云端转存
